项目总结：NS-3 网络仿真 3D 可视化分析工具
一、项目目标（Objective）

构建一个面向 NS-3 仿真的：

可交互、可拖动、支持时间回放的 3D 网络状态可视化分析工具

用于：

网络行为分析（debug）
拥塞定位
算法效果验证（如 LAPS）
二、核心功能需求（Functional Requirements）
1️⃣ 3D 拓扑可视化
从输入文件构建网络拓扑
支持节点与链路显示
支持分层结构展示（多层）即降低其他层的透明度，高亮显示对应层的节点和上下行链路)：
比如
Host
ToR
DU
L1
HRS
但是分层应该在配置文件里面定义，而不是锁死，方便工具复用到其他拓扑，进阶为可以在自由选择看多层的某些子部分的高亮
2️⃣ 交互能力（必须具备）
鼠标拖动（旋转视角）
滚轮缩放
平移视图
点击节点/链路查看详细信息（debug级）
3️⃣ 多指标可视化（可切换）

用户可以选择展示某一类指标：

Queue Length（队列长度）
Sending Rate（发送速率）
Congestion（拥塞状态，如 ECN）

👉 同一时间只显示一种指标（避免信息过载）

4️⃣ 时间回放系统（核心）
基于 NS-3 日志进行离线回放
支持：
播放 / 暂停
时间轴拖动
跳转任意时刻
5️⃣ 动态可视化映射
指标	映射方式
Queue Length	节点颜色（绿 → 黄 → 红）
Sending Rate	链路粗细
Congestion	链路颜色
三、输入数据设计（Data Input）
1️⃣ 拓扑文件（静态）

格式建议（JSON）：

{
  "nodes": [
    {"id": "n1", "layer": "host"},
    {"id": "n2", "layer": "tor"}
  ],
  "links": [
    {"src": "n1", "dst": "n2"}
  ]
}
2️⃣ 状态数据（时间序列）
{
  "timestamps": [0,1,2,...],
  "node_metrics": {
    "n1": {
      "queue": [10,20,15,...]
    }
  },
  "link_metrics": {
    "n1-n2": {
      "rate": [100,200,...],
      "ecn": [0.1,0.3,...]
    }
  }
}
四、系统架构（Architecture）
NS-3 Simulation
      ↓
Log 输出（自定义格式）
      ↓
Python 预处理模块
- 
- 转换为结解析日志构化数据（JSON）
      ↓
前端可视化（Three.js）
- 渲染3D拓扑
- 控制交互
- 时间回放
五、关键设计（Critical Design Decisions）
✔ 1. 使用分层3D布局（核心）

根据网络层级固定Z轴：

层级	Z值
HRS	高
L1	↑
DU	↑
ToR	↑
Host	低

👉 优点：

拓扑结构清晰
易于定位问题层级
✔ 2. 离线回放优先（非实时）
不做实时流式系统
专注：
可重复分析
任意时间跳转
✔ 3. 数据驱动渲染（Data-driven Rendering）
不重新创建对象
只更新：
颜色
大小
粗细
✔ 4. 单指标模式（避免复杂度爆炸）
每次只显示一种 metric
通过 UI 切换
六、技术选型（Tech Stack）
前端（核心）
Three.js（3D渲染）
OrbitControls（交互）
dat.GUI（控制面板）
后端（轻量）
Python（数据预处理）
不需要复杂服务器（初期）
七、分阶段实现计划（Development Phases）
🚀 Phase 1：基础拓扑
渲染节点 + 链路
实现拖动 / 缩放 / 旋转
🚀 Phase 2：单帧数据
加载某一时刻数据
映射 queue → 颜色
🚀 Phase 3：时间系统
时间轴 slider
播放 / 暂停
🚀 Phase 4：多指标切换
queue / rate / congestion
🚀 Phase 5：交互分析
点击节点显示详细信息
高亮选中对象
🚀 Phase 6（可选增强）
Top-K 拥塞高亮
路径可视化
层级过滤
八、非目标（明确边界）

当前不做：

实时仿真可视化（online）
超大规模（>10k节点）
复杂流动画（初期）
九、一句话总结（给AI用）
构建一个基于 Three.js 的 3D 网络可视化系统，输入为 NS-3 仿真日志（经 Python 预处理），支持分层拓扑展示、时间回放、多指标切换（queue/rate/congestion），并具备拖动、缩放、点击分析等交互能力，面向科研级网络行为分析与调试。
