一、拓扑概述
这是一个基于 Pod-Rack-LRS-HRS 架构的数据中心网络拓扑，采用 400Gbps 基础带宽。整体设计遵循"简化机架内互联、强化上行带宽、集中跨 Rack 汇聚"的原则。

二、拓扑规模与层次结构
1. 物理规模
POD 数量：1 个
Rack 数量：4 个（按 1 行 4 列的平面布局排列）
每个 Rack 尺寸：8 行 × 8 列 = 64 个位置
2. 节点分层与数量
第一层：Host（计算节点）
总数：256 个
分布：每个 Rack 包含 8 行 × 8 列 = 64 个 Host
坐标标识：(pod, rack, row, col)，其中 pod=0, rack=0~3, row=0~7, col=0~7
第二层：ToR（机架顶部交换机）
总数：256 个
分布：与 Host 一一映射，每个 (pod, rack, row, col) 位置对应 1 个 ToR
端口配置：
向下端口：1 个（连接 Host）
水平端口：7 个（同列 Mesh）
向上端口：8 个（连接 LRS 层的 DU）
总端口数：16 个
转发延迟：100ns
第三层：LRS（本地汇聚交换机层）
LRS 采用分层架构，每个 Rack 包含 4 个独立的 LRS 层（当然这里仍然可以把四个LRS看成一个LRS有四层）。

3.1 DU（分发单元）
总数：256 个
分布：每个 Rack 有 4 层，每层 16 个 DU
坐标标识：(pod, rack, layer, du_index)，layer=0~3, d~u_index=015
端口配置：
向下端口：8 个（接收来自同一列 8 个 ToR 的连接）
向 L1 端口：4 个（连接 2 个 L1，每个 2 条物理链路）
向 HRS 端口：4 个（连接对应的 HRS）
总端口数：16 个
转发延迟：100ns
3.2 L1（一级汇聚交换机）
总数：64 个
分布：每个 Rack 有 4 层，每层 4 个 L1
坐标标识：(pod, rack, layer, l1_index)，layer=0~3, l1_index=0~3
端口配置：
仅向下端口：16 个（接收 8 个 DU 的连接，每个 DU 用 2 条物理链路）
总端口数：16 个
转发延迟：100ns
特殊说明：L1 不再与任何其他节点连接（无 L1-L1 跨 Rack 连接，无 L1-HRS 连接）
第四层：HRS（高层汇聚交换机）
总数：4 个（全网共享）
坐标标识：hrs_id=0~3
端口配置：
连接端口：256 个（接收来自第 i 层所有 64 个 DU 的连接，每个 DU 用 4 条物理链路）
总端口数：256 个
转发延迟：250ns
核心作用：全网唯一的跨 Rack 通信通道
三、详细连接规则
连接 1：Host → ToR（接入层）
连接模式：严格一一映射
规则描述：坐标为 (pod, rack, row, col) 的 Host 仅连接同坐标的 ToR
链路数量：每个 Host 1 条上行链路
带宽：(16-1)×400Gbps = 6000Gbps（聚合带宽）
延迟：1ns
连接 2：ToR ↔ ToR（Rack 内 Mesh）
连接模式：仅同列 Full Mesh（删除了同行连接）
规则描述：
对于每一列（col=07），该列的 8 个 ToR（row=07）之间两两互连
形成 8 个独立的"列内全连接网格"
同行之间不再有任何直接连接
边数计算（单 Rack）：
每列边数 = C(8,2) = 28 条
8 列总边数 = 8 × 28 = 224 条
带宽：400Gbps
延迟：1ns
连接 3：ToR → DU（上行链路）
连接模式：列绑定 + 双层冗余
规则描述：
第 i 列（i=07）的所有 ToR（共 8 个，row=07）连接到每层 LRS 的 2 个 DU
具体连接：第 i 列 ToR → 每层的 DU[2i] 和 DU[2i+1]
例如：
第 0 列 ToR → 每层的 DU[0] 和 DU[1]
第 1 列 ToR → 每层的 DU[2] 和 DU[3]
...
第 7 列 ToR → 每层的 DU[14] 和 DU[15]
链路数量：
每个 ToR 向上共 8 条链路（4 层 × 每层 2 个 DU）
每条链路为独立物理连接
带宽：400Gbps/条
延迟：5ns
连接 4：DU → L1（LRS 内部汇聚）
连接模式：分组双链路连接
规则描述：
在同一 (pod, rack, layer) 内，16 个 DU 按奇偶编号分为 8 组
对于第 i 组（i=0~7）：
DU[2*i]（偶数编号）连接 L1[0] 和 L1[1]
DU[2*i+1]（奇数编号）连接 L1[2] 和 L1[3]
例如（对于任意一层）：
DU[0] → L1[0]（2 条链路）、L1[1]（2 条链路）
DU[1] → L1[2]（2 条链路）、L1[3]（2 条链路）
DU[2] → L1[0]（2 条链路）、L1[1]（2 条链路）
DU[3] → L1[2]（2 条链路）、L1[3]（2 条链路）
...以此类推
链路数量：
每个 DU 连接 2 个 L1
每个 DU 到每个 L1 之间有 2 条物理链路（edge_count=2，用于带宽翻倍）
每个 DU 总计 4 条出边到 L1 层
带宽：400Gbps/条（实际等效 800Gbps 因双链路）
延迟：1ns
连接 5：DU → HRS（跨 Rack 汇聚）【新增】
连接模式：分层直连
规则描述：
全网 4 个 HRS 按层分配：HRS[i] 专门服务第 i 层 LRS
第 i 层（i=0~3）的所有 DU（无论哪个 Rack）都连接到 HRS[i]
具体连接：
Layer 0 的所有 64 个 DU（4 Rack × 16 个/层）→ HRS[0]
Layer 1 的所有 64 个 DU → HRS[1]
Layer 2 的所有 64 个 DU → HRS[2]
Layer 3 的所有 64 个 DU → HRS[3]
链路数量：
每个 DU 出 4 条物理链路 到对应的 HRS（edge_count=4）
每个 HRS 接收 64 个 DU × 4 条链路 = 256 个端口
带宽：400Gbps/条（实际等效 1600Gbps 因四链路）
延迟：250ns
连接 6：L1 ↔ L1（跨 Rack 互连）【已删除】
改动前：同 layer、同 index 的 L1 在不同 Rack 之间 full mesh
改动后：完全删除，不再有任何 L1 到 L1 的跨 Rack 直接链路
连接 7：L1 → HRS（汇聚接入）【已删除】
改动前：每个 L1 连接所有 2 个 HRS
改动后：完全删除，L1 不再连接任何 HRS
